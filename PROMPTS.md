# Claude Code Prompt 速查表

> 这些是你在手机上常用的 prompt 模板。可以直接复制粘贴，也可以语音输入后微调。

## 🔧 功能开发类

```
开发一个 [功能名称] 功能。要求：
1. [具体要求1]
2. [具体要求2]
确保有错误处理，完成后 commit。
```

```
在 [文件路径] 中添加 [具体功能]，保持现有代码结构不变。
```

```
参考 [某个文件或模块] 的实现方式，开发一个类似的 [新功能]。
```

## 🐛 Bug 修复类

```
[页面/功能] 出现了 [问题描述]。请排查原因并修复。
修复后在 PROGRESS.md 中记录这个 bug 的原因和解法。
```

```
运行 [命令]，如果报错请修复。反复测试直到通过。
```

## 🎨 UI 优化类

```
优化手机端的 [页面/组件] 显示效果：
- 字体太小，增大到适合手机阅读的大小
- 按钮间距太近，增加间距
- 加载时显示 loading 状态
```

## 📝 代码质量类

```
Review 当前代码，找出潜在的 bug 和可优化的地方。
修复问题后更新 PROGRESS.md。
```

```
给 [模块] 添加错误处理和日志记录。
确保所有 subprocess 调用都有 timeout 和异常处理。
```

## 🚀 系统运维类

```
检查系统运行状态，确保：
1. 数据库连接正常
2. 所有 API 端点可访问
3. WebSocket 连接正常
4. 自动备份在正常运行
如有问题请修复。
```

```
优化系统性能：
- 检查是否有内存泄漏
- 优化数据库查询
- 清理过期日志文件
```

## 💡 小技巧

1. **任务描述要具体**：不要说"优化一下"，要说"把任务列表的加载速度优化到 1 秒以内"
2. **一次一个任务**：不要把多个不相关的需求塞到一个 prompt 里
3. **提供上下文**：如果涉及特定文件，在 prompt 中提及文件路径
4. **让它自测**：在 prompt 末尾加上"完成后自行测试，确保功能正常"
5. **沉淀经验**：重要任务完成后加一句"更新 PROGRESS.md"
